# 台灣股票分析器（AI）實作藍圖

## 1) 目標
建立一個針對台灣股市的 AI 分析系統，能夠：
- 持續學習股票知識與市場結構。
- 每天固定在早上與中午提供候選股票清單。
- 對每檔標的輸出：
  - 推薦原因
  - 預估漲幅區間
  - 風險等級與期望報酬
- 當預測與實際表現不符時，啟動「失敗檢討與再訓練」流程。

## 2) 系統核心原則
1. **可驗證性**：每次推薦都要可追蹤模型版本、特徵快照、推論時間。
2. **風險優先**：不是只看漲幅，必須同時輸出最大回撤與停損建議。
3. **自我修正**：績效低於門檻時，自動觸發誤差分析與再訓練。
4. **嚴禁保證獲利**：僅提供機率性判斷，不宣稱 100% 準確。

## 3) 資料管線（Data Pipeline）
- 市場資料：OHLCV、成交量、分時資料（可用時）。
- 基本面資料：營收、EPS、毛利率、現金流、估值比率。
- 籌碼與事件：法人買賣、融資融券、重大訊息、除權息。
- 衍生特徵：
  - 技術指標（RSI, MACD, 均線乖離, ATR）
  - 波動與流動性（年化波動、換手率）
  - 市場 regime（多頭/空頭/盤整）

建議資料切分：
- 訓練集：歷史長區間
- 驗證集：時間序列滾動驗證
- 測試集：最近區間（不可洩漏未來資訊）

## 4) 模型架構（建議）
採用多模型集成（Ensemble）：
- 主模型：LightGBM / XGBoost（表格特徵表現穩定）
- 輔助模型：LSTM / Temporal Transformer（時間序列）
- 校準層：Platt scaling / isotonic regression（機率校準）

輸出設計：
- `P(up_5d)`：未來 5 日上漲機率
- `ExpectedReturn_5d`：未來 5 日期望報酬
- `RiskScore`：風險分數（0-100）
- `Confidence`：信心度（需經校準）

## 5) 推薦邏輯（每天早上/中午）
排程時間（台北時間）
- 早上：09:00
- 中午：12:00

篩選流程：
1. 流動性過濾（成交量、價差）
2. 排除重大風險事件（財報雷、異常波動）
3. 以綜合分數排序：
   - `Score = w1*P(up_5d) + w2*ExpectedReturn_5d - w3*RiskScore`
4. 產生 Top-N 名單，並附理由與風險提醒

每檔輸出範本：
- 股票代號 / 名稱
- 推薦理由（3 點）
- 預估漲幅區間（如 +2% ~ +6%）
- 停損建議（如 -3%）
- 失效條件（例如跌破某均線或成交量異常）

## 6) 失敗重學習機制（Closed-Loop Learning）
定義失敗條件（可同時使用）：
- 命中率連續 N 天低於門檻
- 實際報酬顯著低於基準（如大盤）
- 校準誤差惡化（Brier score 上升）

觸發後流程：
1. 失敗案例收集（推薦時點、特徵、結果）
2. 分群分析（產業別、波動 regime、事件類型）
3. 找出缺點（特徵不足、過擬合、延遲）
4. 更新特徵與超參數
5. 重新訓練 + 回測 + 穩定性測試
6. 達標才升級生產版本（Champion/Challenger）

## 7) 評估指標（必看）
- 分類：AUC、F1、Precision@K
- 回歸：MAE、RMSE（預估漲幅）
- 投資績效：
  - 年化報酬
  - 最大回撤（MDD）
  - Sharpe / Sortino
  - 勝率與盈虧比
- 穩定性：不同年份/行情 regime 的一致性

## 8) 風控與合規提醒
- 強制倉位上限（單檔、單日、單產業）。
- 強制停損、停利、最大回撤保護。
- 明確標註「非投資保證，僅供研究參考」。
- 避免使用來源不明與違規取得資料。

## 9) MVP（最小可行版本）里程碑
- 週 1-2：資料蒐集與清理、特徵工程基礎版
- 週 3-4：建立首版模型與回測框架
- 週 5：早/中午排程與報表輸出
- 週 6：失敗重學習流程 + 監控儀表板

## 10) 你可以先落地的下一步
1. 先定義「預測週期」（3 日、5 日或 10 日）。
2. 決定基準指數與績效門檻。
3. 建立第一版資料字典與特徵清單。
4. 先上線保守策略（低風險 + 高流動性）。
5. 連續運行 4 週後再擴大標的池。
